Connect a GitHub Repo — Setup Wizard

Follow these five steps to connect any GitHub repository to CodeSheriff. Each new repo you want to review needs its own webhook pointing at this service.

1
Enter the GitHub repo you want to review

Type the repository in owner/name format — used to build links in steps below.

2
Expose the webhook service to the internet

GitHub needs a public HTTPS URL to deliver webhook events. The webhook service runs on port 8001 locally. Use ngrok (or any tunnel) to expose it:

ngrok http 8001

Then paste the https://…ngrok.io URL ngrok gives you:

3
Generate a webhook secret

This shared secret lets CodeSheriff verify that payloads really come from GitHub (HMAC-SHA256 signature check). Generate a fresh one — keep it secret.

Add this to your .env file (then restart the services):

WEBHOOK_SECRET=<generate above first>
4
Add the webhook on GitHub

Open your repo's webhook settings and fill in the fields shown below. The link auto-updates once you enter a repo in Step 1.

→ Open GitHub Webhook Settings

Payload URL
<enter public URL in Step 2>
Content type
application/json
Secret (paste the value from Step 3)
Events Choose Let me select individual events → tick Pull requests only
Active ☑ checked
5
Test the full pipeline

Open a real PR on the repo (or use an existing one) and run the demo script to fire a simulated webhook — no actual PR needed locally:

python test_demo.py # enter repo in Step 1 to fill this in

Or run a dry-run that tests only webhook reception + DB logging (no GitHub API calls required):

python test_demo.py --dry-run

Once a review completes, it appears in the Recent Reviews table on this dashboard — auto-refreshes every 30 seconds.
